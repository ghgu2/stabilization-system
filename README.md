# Мини-система стабилизации камеры

## Описание

В данном репозитории был реализован пайплайн стабилизации камеры на основе **плотного оптического потока Farnebäck** и **фильтра Калмана**. Код принимает видео с дрожью камеры и формирует стабилизированную версию с компенсацией дрожания.

## Выбранные методы

| Этап | Метод | Обоснование |
|---|---|---|
| Оптический поток | **Farnebäck** (плотный) | Считает смещение в каждом пикселе, а не только в отдельных точках, тем самым давая более полную картину движения |
| Сглаживание траектории | **Фильтр Калмана** | Данный фильтр учитывает инерцию камеры, а также лучше отделяет намеренное панорамирование от дрожи лучше |
| Компенсация движения | Аффинная модель (dx, dy, da) + `cv2.warpAffine` | Простая и быстрая модель, достаточная для компенсации сдвигов и поворотов |
| Постобработка | Crop 7% + resize | Убирает краевые артефакты от которые появляются после предыдущих этапов |

## Пайплайн

1. **Импорт видео** — загружаем видео в память.
2. **Оценка движения с помощью Farnebäck** — для каждой пары кадров в видео вычисляется плотный оптический поток, и глобальное движение оценивается через медиану поля смещений  и приближенный угол поворота.
3. **Построение траектории** — интеграция межкадровых смещений.
4. **Сглаживание фильтром Калмана** — фильтр сглаживает каждую компоненту траектории (dx, dy, da) отдельно.
5. **Компенсация движения** — разница между сглаженной и реальной траекторией применяется как аффинное преобразование (`warpAffine`) к каждому кадру.
6. **Crop + resize** — обрезка 7% краёв и resize до исходного разрешения.
7. **Сохранение** — сохранение стабилизированного видео в формате mp4.

## Визуализации

- **Траектории камеры** — графики по dx, dy, da: оригинальная vs сглаженная Калманом.
- **Сравнение «до / после»** — 5 различных кадров из видео со сравнением.
- **Примеры warping-а** — оригинал vs результат warpAffine с параметрами коррекции.
- **Поле оптического потока** — стрелки поверх кадра + HSV-карта, на которой цвет это направление, а яркость это скорость.
- **Метрики** — с различным сравнением оригинал vs стабилизированная картинка.

## Проблемы стабилизации

Основные источники ошибок стабилизации:

- **Motion blur** — на быстрых движениях Farnebäck дает неточное поле потока.
- **Панорамирование** — видео снято с постоянным движением камер из-за чего фильтр Калмана с высоким R отделяет дрожь от намеренного движения.
- **Малая текстура** — однородные области, например, газон и небо дают шумное поле потока.
- **Crop + resize** — данные результаты приводят к тому, что мы получаем изображение с меньшим числом пикселей, а затем трансформируем в исходное разрешение, и это приводит к слегка размытой картинке.
- **Параллакс** — аффинная модель не учитывает глубину сцены, из-за этого объекты на разном расстоянии (газон vs деревья) движутся по-разному, что вносит ошибку в медиану потока.
